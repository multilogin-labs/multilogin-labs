# Migrate from AdsPower → Multilogin X

> [Partner pricing →](https://multilogin.com/pricing/?utm_source=saas&utm_medium=partner&a_aid=saas&a_bid=f5fad549) · **`SAAS50`** · **`MIN50`**

A 5-minute migration from AdsPower CSV exports to Multilogin X profile JSON.

## 1. Export from AdsPower

AdsPower → **Account Center** → **Export Profiles** → CSV (UTF-8).

Common columns: `id`, `name`, `os`, `browser`, `proxy_type`, `proxy_host`, `proxy_port`, `proxy_user`, `proxy_pass`, `group`, `notes`.

## 2. Convert with one command

```bash
node tools/migrate-adspower.mjs \
  --input ads.csv \
  --output mlx.json \
  --folder $MULTILOGIN_FOLDER_ID
```

Sample output ([test fixture](../../tools/migrate-adspower.test.csv)):

```json
[
  {
    "name": "acme-fb-1",
    "folder_id": "...",
    "os_type": "win",
    "browser_type": "mimic",
    "proxy": { "type": "http", "host": "1.2.3.4", "port": 8080, "username": "alice", "password": "pw1" },
    "notes": "Migrated from AdsPower\nid=1\ngroup=facebook\nnotes=blue team"
  }
]
```

## 3. Bulk-create in Multilogin X

```bash
# Python
python examples/02-python-bulk-create/bulk_create.py mlx.json
```

## What gets mapped

| AdsPower | Multilogin |
|---|---|
| `os` `Win11/macOS` | `os_type` `win/macos` |
| `browser` `Mimic/Stealthfox` | `browser_type` `mimic/stealthfox` |
| `proxy_*` | `proxy.{type,host,port,username,password}` |
| Other columns | Concatenated into `notes` so nothing is lost |

## Caveats

- Cookies and storage are **not** migrated (different vendor formats). Use the [09-import-cookies recipe](../api/cookbook/09-import-cookies.md) per profile.
- Fingerprint settings (canvas, WebGL, fonts) are regenerated by Multilogin — that's the whole point of switching.
- Validate a few profiles by hand before running the full import.
